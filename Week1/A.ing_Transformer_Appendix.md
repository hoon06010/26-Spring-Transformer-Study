# A.ing Transformer Appendix

## 순차 데이터와 언어 모델링의 발전

### 1 연속적인 데이터와 토크나이징 (Tokenizing) & 임베딩 (Embedding)

왜 컴퓨터는 문장을 통째로 이해하지 못하고 잘게 쪼개야 할까?

자연어는 시간의 흐름에 따라 의미가 누적되는 '시계열 데이터(Sequential Data)'입니다. 하지만 컴퓨터는 텍스트를 있는 그대로 연산할 수 없고 숫자(벡터)로 변환된 형태만 처리할 수 있습니다. 토크나이징은 텍스트를 의미를 가진 최소 단위로 나누는 작업입니다. 이 과정을 거쳐야 각 토큰을 고유한 숫자로 매핑할 수 있습니다.

왜 토큰화한 단어에 그냥 단순한 번호를 매기지 않고 '임베딩(Embedding)'이라는 복잡한 과정을 거칠까?

텍스트를 토큰으로 쪼갠 뒤, 단순히 '사과=1, 바나나=2'처럼 번호표를 부여하거나 0과 1로만 이루어진 원-핫 벡터(One-hot vector)로 만들면, 컴퓨터는 단어 간의 '의미적 연관성'을 전혀 알 수 없습니다. 사과와 바나나가 과일로서 비슷하다는 것을 수치상으로 증명할 길이 없는 것입니다.
임베딩은 단어들을 수많은 차원의 '밀집 벡터(Dense Vector)' 공간에 흩뿌려 배치하는 기술입니다. 이를 통해 같이 자주 쓰이는 단어들이나 문맥이 비슷한 단어들(예: $King - Man + Woman \approx Queen$)은 공간상에서 서로 가까운 거리에 놓이게 되어, 컴퓨터가 마침내 단어의 '뉘앙스와 맥락'을 수학적으로 계산할 수 있게 됩니다.

### 2 RNN의 동작 방식 (Recurrent Neural Network)

왜 기존의 일반 신경망(FNN, CNN)으로는 언어를 처리하기 어려울까?

일반적인 인공신경망은 입력 데이터가 서로 독립적이라고 가정하며, 입력 길이도 고정되어 있습니다. 하지만 문장은 앞 단어가 뒤 단어에 영향을 미치는 구조적 특징을 가집니다.

그래서 RNN은 내부에 '은닉 상태(Hidden State, 기억)'를 도입했습니다. 현재 시점의 입력값과 이전 시점의 은닉 상태를 함께 연산하여 현재의 은닉 상태를 갱신합니다. 이를 통해 문맥(Context) 정보를 시간의 흐름에 따라 순차적으로 전달합니다.

### 3 다음 토큰 예측 (Next Token Prediction, NTP)

왜 언어 모델은 '다음 단어 맞추기' 방식으로 학습할까? (GPT와 같은 언어모델이 주로 학습하는 방식이고 BERT는 다른 방식으로 학습합니다.)

언어 모델의 본질은 주어진 문맥 다음에 등장할 단어의 확률을 최대화하는 것입니다. 문장의 앞부분을 주고 다음 단어를 예측하게 하는 과정 자체를 반복하면, 사람이 일일이 정답을 매겨줄 필요 없이 방대한 텍스트 데이터만으로 자율학습(Self-supervised learning)이 가능해집니다.

왜 단순한 '다음 단어 맞추기'가 언어의 문법과 지식까지 학습하게 만들까?

NTP(Next Token Prediction)는 이전 단어들($w_1, w_2, \dots, w_{t-1}$)이 주어졌을 때 다음 단어($w_t$)가 무엇일지 어휘 사전 내 모든 토큰의 확률 분포를 계산하는 통계적 과정입니다.
예를 들어 "대한민국의 수도는 [ ]"라는 문맥이 주어지면, 모델은 "서울"이라는 토큰에 가장 높은 확률을 할당하도록 내부 가중치를 조정합니다. 이 단순한 확률 예측 과정이 수억, 수조 번의 거대한 텍스트 코퍼스 위에서 반복되면, 모델은 오차를 줄이기 위해 필사적으로 데이터 내부의 '패턴'을 찾아야 합니다.
결과적으로 모델은 단순한 단어 빈도수뿐만 아니라 "주어 뒤에는 동사가 온다"는 문법적 구조, "사과와 배는 과일이다"라는 의미적 관계, 나아가 세계에 대한 객관적 팩트(지식)까지 숫자(가중치)의 형태로 파라미터 공간 내부에 압축하여 저장하게 됩니다.

### 4 RNN의 구조적 한계

왜 문장이 길어지면 RNN의 성능은 급격히 하락할까?

가장 치명적인 문제는 장기 의존성(Long-Term Dependency) 문제입니다. RNN은 구조상 역전파(Backpropagation) 시 시점을 거슬러 올라가며 기울기를 반복해서 곱해야 합니다. 문장이 길어지면 1 미만의 값이 계속 곱해지면서 기울기가 0에 수렴해버리는 '기울기 소실(Vanishing Gradient)' 현상이 발생하여, 문장 초반의 중요한 정보가 문장 끝까지 도달하기도 전에 잊혀지게 됩니다.

왜 역전파 과정에서 하필 '1 미만의 값'이 계속 곱해진다는 것일까?

이는 RNN 내부의 활성화 함수(Activation Function)와 가중치(Weight)의 수학적 특성 때문입니다.

활성화 함수의 한계: RNN은 비선형성을 부여하기 위해 보통 Sigmoid나 Tanh 함수를 사용합니다. 모델을 학습시키기 위해 미분(기울기 계산)을 수행할 때, Sigmoid 함수의 미분 최댓값은 0.25에 불과하며, Tanh 역시 최댓값이 1입니다. 즉, 층(시점)을 거칠 때마다 기울기 값은 필연적으로 1보다 작아지거나 유지될 수밖에 없습니다.

연쇄 법칙(Chain Rule): 역전파는 출력단에서 발생한 오차 신호를 과거 시점으로 전달하기 위해 각 시점의 미분값과 가중치를 계속해서 곱해 나가는 연쇄 법칙을 따릅니다.
만약 미분값과 가중치가 곱해진 결과가 0.9라고 가정해 봅시다. 문장 길이가 10단어만 되어도 $0.9^{10} \approx 0.34$로 깎이고, 50단어가 되면 $0.9^{50} \approx 0.005$로 순식간에 0에 수렴해버립니다. 결국 50번째 전의 단어가 현재 단어에 미치는 영향력(기울기)이 0.005로 소멸하여, 모델은 멀리 떨어진 단어의 맥락을 전혀 학습할 수 없게 됩니다.

### 5 RNN의 여러 가지 유형

왜 한 가지 RNN 구조만으로 모든 NLP 문제를 풀 수 없을까?

우리가 풀고자 하는 현실의 문제마다 입력과 출력의 형태가 다르기 때문입니다.

Many-to-One: 다수의 토큰을 입력받아 하나의 결과를 출력합니다. (예: 긍정/부정 감성 분류, 스팸 필터링)

One-to-Many: 하나의 입력(이미지 등)을 받아 다수의 토큰(문장)을 출력합니다. (예: 이미지 캡셔닝)

Many-to-Many: 다수의 토큰을 받아 다수의 토큰을 출력합니다. (예: 기계 번역, 챗봇)

### 6 Seq2seq 개념 및 문제점

왜 번역과 같이 입력과 출력의 길이가 다른 작업에는 새로운 구조가 필요했을까?

'I love you'를 '나는 너를 사랑해'로 번역할 때, 영어(3단어)와 한국어(4단어)는 길이가 다르고 어순도 다릅니다. 이를 해결하기 위해 입력 문장을 전부 읽어 하나의 벡터로 압축하는 **인코더(Encoder)**와, 이 압축된 벡터를 바탕으로 출력 문장을 하나씩 생성하는 디코더(Decoder) 구조를 분리한 Seq2seq가 등장했습니다.

문제점: 인코더가 아무리 긴 문장을 읽어도 최종적으로 **'하나의 고정된 크기의 컨텍스트 벡터(Context Vector)'**에 모든 정보를 욱여넣어야 한다는 치명적인 병목 현상(정보 손실)이 발생합니다.

### 7 Attention: 시점마다 다른 Context Vector의 사용

왜 디코더가 번역할 단어를 고를 때마다 인코더의 모든 정보를 똑같이 참조해야 할까?

사람이 번역할 때 현재 번역하는 단어와 관련 깊은 원문 단어에 '집중(Attention)'하는 것처럼, 모델도 그래야 합니다. 디코더가 단어를 예측하는 매 시점마다, 인코더의 전체 출력 중 현재 예측에 꼭 필요한 부분에만 가중치를 주어 '유동적인 맞춤형 컨텍스트 벡터'를 만듭니다.

### 7.1 Attention: Context Vector 만들기

왜 Query, Key, Value라는 용어를 쓰며, 실제 자연어 번역에서는 어떻게 작동할까?

기계 번역 모델이 "I love you"를 "나는 너를 사랑해"로 번역하는 상황을 예시로 들어보겠습니다. 현재 디코더가 "나는", "너를"까지 출력했고, 이제 "사랑해"를 번역해야 할 차례입니다.

Query (질의): 디코더의 현재 상태.
"나는 지금 '너를' 다음 단어를 만들어야 해. 원문의 어떤 정보가 필요할까?"

Key (키): 인코더에 입력된 원문 단어($w_1, w_2, w_3$)들이 각자 들고 있는 이름표.

$K_1$: "나는 '나(I)'에 대한 정보야."

$K_2$: "나는 '사랑(love)'에 대한 정보야."

$K_3$: "나는 '너(you)'에 대한 정보야."

Value (값): 인코더의 원문 단어들이 가지고 있는 실제 의미(벡터 알맹이).

디코더(Query)는 다음 단어를 잘 예측하기 위해 원문의 각 단어 이름표(Key)와 자신을 내적하여 **유사도(관련성)**를 측정합니다. 그 결과 "I(0.1)", "love(0.8)", "you(0.1)"라는 가중치(Attention Score)를 얻게 됩니다. 디코더는 이 가중치를 바탕으로 "love"가 가진 실제 의미(Value)를 가장 많이 끌어와 맞춤형 Context Vector를 만들고, 이를 통해 "사랑해"라는 단어를 정확히 출력해 냅니다.

### 7.2 Attention의 학습 원리와 해석

왜 어텐션은 해석이 가능할까?

어텐션 가중치는 모델이 오차를 줄이는 방향으로 역전파를 수행하며 자동으로 학습됩니다. 결과적으로 어떤 출력 단어를 만들 때 어떤 입력 단어를 강하게 참조했는지(Alignment) 숫자로 확인할 수 있어, 블랙박스였던 딥러닝 모델의 판단 근거를 시각화하고 해석할 수 있게 되었습니다.

### 7.3 RNN+Attention의 두 가지 문제점

어텐션으로 성능은 올랐지만 왜 여전히 한계가 있을까?

어텐션이 추가되었어도 전체 구조의 뼈대가 여전히 RNN이기 때문에 다음과 같은 두 가지 물리적인 제약이 따릅니다.

연산 속도와 병렬화 불가 (순차적 연산의 병목):
RNN의 가장 큰 특징은 $t$ 시점의 은닉 상태를 계산하려면 반드시 $t-1$ 시점의 연산이 완전히 끝나야 한다는 점입니다. 만약 100단어짜리 문장이라면 100번의 스텝을 순서대로 하나씩 밟아야 합니다. 현대의 GPU는 수천 개의 코어를 가지고 방대한 행렬을 동시에 계산(병렬 처리)하는 데 특화되어 있지만, RNN 구조는 이전 단어의 처리가 끝날 때까지 수많은 코어가 놀면서 대기해야 하므로 하드웨어의 자원을 낭비하게 되고 학습 속도가 매우 느립니다.

정보 전달 경로의 길이 (여전히 남아있는 장기 의존성):
어텐션을 통해 디코더가 인코더의 특정 단어를 직접 참조(지름길)할 수 있게 되었지만, 인코더가 처음에 원문 단어들의 상태(Hidden State)를 생성할 때는 여전히 RNN의 순차적 경로를 통과해야 합니다. 즉, 첫 번째 단어의 의미가 백 번째 단어의 상태를 계산할 때 반영되려면 여전히 100번의 층(연산)을 거쳐야 합니다. 전달 경로가 물리적으로 길기 때문에 정보의 희석, 변질, 기울기 소실 문제가 완전히 사라지지 않고 잔존합니다.

### 8 트랜스포머의 Self-Attention

왜 굳이 느리고 비효율적인 RNN을 유지해야 할까? 아예 빼버리면 안 될까?

이 질문에서 출발한 것이 트랜스포머(Transformer)입니다. 순차적 처리(RNN)를 완전히 버리고, 입력 문장 내의 모든 단어들이 서로가 서로에게 어텐션을 수행하는 구조만으로 문맥을 파악합니다.

왜 이름이 그냥 Attention이 아니라 'Self'-Attention일까?

Seq2seq의 기존 Attention은 번역을 할 때 '디코더(출력 문장)'가 '인코더(입력 문장)'를 바라보는 구조였습니다. 두 주체가 달랐습니다.
반면 Self-Attention은 입력 문장 내의 단어들이 '자기 자신들끼리(Self)' 서로를 바라보며 문맥을 파악합니다. 예를 들어 "The animal didn't cross the street because it was too tired" 라는 문장에서, 컴퓨터는 'it'이 animal인지 street인지 알 수 없습니다. 이때 Self-Attention을 통해 문장 내부의 단어들끼리 유사도(Attention Score)를 계산하면, 'it'이 'animal'과 매우 강하게 연결되어 있다는 것을 스스로 파악할 수 있게 됩니다.

트랜스포머가 단어를 동시에 병렬 처리한다면, 단어의 '순서(어순)'는 어떻게 파악할까?

"나는 밥을 먹고 공부를 했다"와 "나는 공부를 하고 밥을 먹었다"는 완전히 다른 문장입니다. 기존 RNN은 단어를 하나씩 순서대로 읽기 때문에 자연스럽게 어순을 알았지만, 트랜스포머는 모든 단어를 한꺼번에 병렬로 연산하기 때문에 문맥은 빨리 파악해도 단어의 물리적인 순서 정보는 잃어버리게 됩니다.

이를 해결하기 위해 트랜스포머는 입력 단어 벡터들에 **'위치 인코딩(Positional Encoding)'**이라는 꼬리표를 수학적으로 더해줍니다. "이 단어는 첫 번째야", "이 단어는 세 번째야"라는 고유의 위치 정보를 사인(Sine)과 코사인(Cosine) 함수로 만들어 벡터에 더해줌으로써, 모델이 병렬 처리를 하면서도 문장의 어순을 정확히 인식할 수 있게 만듭니다.
